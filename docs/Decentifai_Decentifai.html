Class: Decentifai

Decentifai(options)

new Decentifai(options)

Create a new Decentifai instance, a single entity to manage all P2P connections and learning.
Parameters:
Name Type Description
options Object Configuration options for Decentifai
Properties
Name Type Description
roomId string Unique identifier for the federation
backend string Framework backend for the model (only supports 'tfjs', 'onnx' and 'generic', but 'onnx' support is not natively available yet)
model Object The actual model in the case of TF.js; an object containing paths to the checkpoint state and the training, optimizer and evaluation models in the case of ONNX; or an object containing train, test, extractParameters and updateParameters functions.
Properties
Name Type Description
train function Function to train the model. Optional in case of a TF.js model where trainingData is supplied.
test function Function to test the model.
extractLocalParametersFunc function Function to extract parameters from the model (default provided for TF.js models)
updateLocalParametersFunc function Function to update model with new parameters (default provided for TF.js models)
trainingData Object | Array | function Local training dataset, or a function that returns training data for a given round
testData Object | Array | function Local test dataset, or a function that returns test data
trainingOptions Object Options to pass to the model during training
federationOptions Object Options corresponding to how the federation should be conducted.
Properties
Name Type Description
password String Password that would be required to join the federation.
aggregationMethod String Aggregation strategy to be used for parameter aggregation between rounds (optional, defaults to 'federatedAveraging').
aggregateParametersFunc function Custom aggregation function to be used in place of aggregationMethod.
minPeers Number Minimum number of connected peers to reach quorum
maxPeers Number Maximum number of connected peers to be allowed in the federation.
minRounds Number Minimum training rounds to run the federation for.
maxRounds Number Maximum training rounds to run the federation for.
waitTime Number Wait time (in milliseconds) between federation actions, in order to account for network asynchronicity between peers.
convergenceThresholds Object Thresholds to check for model convergence.
Properties
Name Type Description
parameterDistance Number RMS distance of model parameters across stability window before convergence
lossDelta Number Change in loss across stability window before convergence should be claimed
accuracyDelta Number Change in accuracy across stability window before convergence can be claimed
stabilityWindow Number Window size of consecutive rounds to check if convergence thresholds are met
autoTrain boolean Whether to automatically manage training rounds
signaling Array Array of signaling server URLs
metadata Object Additional metadata about this peer
iceServers Array ICE servers to be used for WebRTC connections.
debug boolean Enable debug logging
